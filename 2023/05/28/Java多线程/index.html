多线程

线程与任务

  • 线程和任务的关系：脱离了任务的线程是没有意义的
  • 线程对象是通过Thread创建的
  • 任务是通过Runnable创建的

三种实现方式 1. 继承Thread类（不是接口） 2. 实现Runnable接口，本质是代理模式（Thread和） 3. 实现Callable

public new Thread(Runnable target)

流程

  1. 创建Tread对象，同时传入任务
  2. 启动线程（并不是立刻执行，需要等待CPU资源）
  3. 一旦拿到资源，开始执行任务，拿到资源后，调用Runnable的run方法(在Thread的run方法中)

实现方法对比

  1. 继承Thread类 直接重写run方法，缺点是任务与线程的耦合度太高，不太灵活
  2. 实现接口的形式 通过传参的形式传入任务，成功解耦合
  3. 使用匿名内部类，简化代码
new Thread(new Runnable(){
     @Override
    public void run() {
        for (int d = 20; d > 0; d--) {
            System.out.println(d);
        }
    }
})
  1. sleep方法 sleep()不在于谁调用，而是在在于写在哪里，写在哪里就是哪个线程休眠
  2. yield方法 yield()方法不一定成功，还是得看CPU如何调度
  3. join方法 合并线程，插队，必须等待join执行结束后其余线程才可以执行，可能会使得线程阻塞，使用方法thread.join(),thread是实例化的对象

解决多线程并发问题

参考文章 1. 多线程并发问题：多个程序并发执行时，因共享资源被多个线程访问导致的程序执行不稳定，不可预测的问题 2. 解决方法 - synchronized关键字，放在代码块上，保证同一时刻只有一个线程调用这段代码块 - lock锁。Lock接口是Java提供的另一种线程同步机制，相比synchronized关键字，Lock接口提供了更加灵活的锁机制，可以实现更加细粒度的线程同步 - volatile。保证变量在多个线程直接共享值

常见的用于多线程的类

Future、Runnable、FutureTask，其中FutureTask实现Future和Runnable接口

  • Callable是用于产生结果(execuator.submit(callable))，Future是用于获取结果的（future.get()，调用会阻塞当前进程），同时也可以取消运行（cancel）
  • FutureTask可以作为参数传入到execuator的参数,也可以作为future利用get获取结果
  • Callable是任务，Future是任务的结果

ThreadLocal用法

  • 主要作用：保证变量再线程之间的隔离，解决线程安全问题。
  • 应用场景：全局共享用户信息，替代重入方法的参数的显示传递，解决线程安全的问题
  • 存在问题：可能存在内存泄漏的问题

Java线程池的线程保活和回收

  • 核心线程保活：通过阻塞队列的方式来保活，当阻塞队列中没有任务时，线程调用poll会导致线程阻塞，如果在一定时间内阻塞队列中还未有任务，则会判断当前线程是否是核心线程，是则重新创建一个线程，否则就让线程正常执行完毕后退出。此外，如果线程发送异常导致线程退出，使用addworker方法新建一个线程。

  • 非核心线程回收：

  • 阻塞队列：当队列满时，调用插入方法的线程会被阻塞，队列为空时，调用弹出方法的线程会被阻塞

AQS

  • 定义：用于控制共享资源访问的同步框架，在其上实现的同步器有ReentrantLock、Sempaphore等。
  • 原理：如果共享资源没空闲，则将当前请求的资源的线程设置为占有共享资源的线程，否则就需要定义线程阻塞等待和唤醒的机制，以及公平锁和非公平锁的实现。在源码层面用state来标识共享资源的状态（CAS），利用CHL的变体双向链表来存放等待的线程。在API层面上提供tryAcquire和tryRelease来获取和释放共享资源，tryAcquireShared、tryReleaseShared
